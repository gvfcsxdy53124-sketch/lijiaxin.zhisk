# 知识库文件上传 PRD

## 1. 文档信息

- 产品模块：知识库 / 文件上传入库
- 覆盖范围：上传文件 Step 1、规则配置 Step 2、文件处理 Step 3
- 当前版本：V1.0
- 目标读者：产品、设计、前端、后端、测试

## 2. 背景与目标

知识库需要支持多种来源的数据进入系统，并在入库前完成解析、标签、切片、索引等配置。不同上传方式的数据结构不同，因此规则配置不能使用同一套表单，需要根据上传方式展示对应配置。

本模块目标：

- 支持 5 种上传方式：文本文档、表格型知识、图片文件、音频文件、网页读取。
- Step 1 完成数据来源选择与文件/网页地址添加。
- Step 2 根据上传方式展示不同规则配置。
- Step 3 展示文件处理进度和最终规则摘要。
- 网页读取即使真实抓取失败，也必须生成可预览内容，保证完整流程可走通。
- 标签作为所有上传方式的通用能力，用于后续分类、检索过滤和召回增强。

## 3. 用户角色

- 知识库管理员：创建知识库、上传文件、配置入库规则、管理标签。
- 普通成员：根据权限查看知识库内容，通常不参与上传配置。
- 系统管理员：拥有所有页面和知识库操作权限，可配置角色、成员和知识库权限。

## 4. 入口与前置条件

入口：

- 进入知识库模块。
- 选择某个知识库详情。
- 点击上传文件入口，进入三步上传流程。

前置条件：

- 用户拥有当前知识库的上传权限。
- 当前知识库处于可编辑或可管理状态。
- 若知识库为部分公开，上传后的文件继承知识库访问范围，后续可叠加文件级标签。

## 5. 总体流程

```text
Step 1 上传文件
  -> 选择上传方式
  -> 添加本地文件或网页地址
  -> 校验上传/抓取状态

Step 2 规则配置
  -> 展示上传方式摘要
  -> 添加文件标签
  -> 根据上传方式展示对应规则配置
  -> 支持预览块/解析预览
  -> 配置文本预处理和索引方式

Step 3 文件处理
  -> 展示处理状态
  -> 展示规则摘要
  -> 展示文件处理进度
  -> 处理完成后生成文档记录
```

## 6. 核心交互逻辑

### 6.1 页面进入与初始化

用户点击知识库详情页的上传入口后，系统打开上传流程页面。

进入页面时初始化以下状态：

| 状态 | 初始值 | 说明 |
| --- | --- | --- |
| 当前步骤 | Step 1 | 默认进入上传文件步骤 |
| 上传方式 | 导入文本文档 | 默认选中第一个上传方式 |
| 上传文件列表 | 空 | 尚未添加文件 |
| 网页地址列表 | 空 | 尚未添加网页 |
| 文件标签 | 空 | 尚未添加标签 |
| 切片方式 | 智能切片 | 文本文档、音频、网页默认规则 |
| 预览内容 | 空 | 进入 Step 2 后按需生成 |
| 处理状态 | idle | 未开始处理 |

若用户从流程中取消：

- 如果未上传文件、未添加网页、未添加标签，直接关闭上传流程。
- 如果已有文件、网页或标签，弹出退出确认。
- 用户确认退出后，清空当前上传流程临时状态。
- 用户取消退出后，保持当前页面状态不变。

### 6.2 Step 1 上传方式切换

用户点击不同上传方式卡片时：

1. 当前上传方式立即切换。
2. 上传区域文案、支持格式、输入控件同步变化。
3. 已添加的本地文件或网页地址不自动跨类型复用。
4. 切换上传方式后，下一步按钮根据当前方式重新计算可用状态。

上传方式切换规则：

| 用户操作 | 系统行为 |
| --- | --- |
| 点击导入文本文档 | 展示文档拖拽上传区 |
| 点击导入表格型知识 | 展示表格拖拽上传区 |
| 点击导入图片文件 | 展示图片拖拽上传区 |
| 点击导入音频文件 | 展示音频拖拽上传区 |
| 点击读取网页数据 | 展示网页 URL 输入表单 |

### 6.3 Step 1 本地文件上传交互

本地文件支持点击上传和拖拽上传。

点击上传：

1. 用户点击上传区域。
2. 系统打开文件选择器。
3. 用户选择文件。
4. 系统校验文件格式。
5. 文件加入上传列表，状态为 uploading。
6. 上传进度从 0% 开始增长。
7. 上传成功后状态变为 success。
8. 上传失败后状态变为 error。

拖拽上传：

1. 用户将文件拖入上传区域。
2. 上传区域出现高亮态。
3. 用户释放文件。
4. 文件加入上传列表。
5. 后续状态与点击上传一致。

文件列表交互：

| 操作 | 系统行为 |
| --- | --- |
| 删除文件 | 从上传列表移除该文件 |
| 删除失败文件 | 若列表中无其他失败文件，关闭“只看问题文件”状态 |
| 查看问题文件 | 只展示 status 为 error 的文件 |
| 分页切换 | 更新当前文件列表页码 |

下一步按钮启停：

| 条件 | 下一步状态 |
| --- | --- |
| 文件列表为空 | 禁用 |
| 存在上传中文件 | 禁用 |
| 存在失败文件 | 禁用 |
| 至少一个文件 success 且无 uploading/error | 启用 |

### 6.4 Step 1 网页读取交互

网页读取不走本地文件上传，而是通过 URL 创建网页虚拟文件。

添加链接流程：

1. 用户输入网页 URL。
2. 点击“添加链接”。
3. 若 URL 为空，系统提示“请输入网页链接”。
4. URL 非空时，系统创建网页地址记录。
5. 系统同步创建一条网页虚拟文件。
6. 网页地址状态进入 fetching。
7. 网页虚拟文件状态进入 uploading。
8. 系统尝试抓取网页 HTML 正文。
9. 若抓取成功，网页地址状态变为 success，网页虚拟文件状态变为 success。
10. 若抓取失败，系统生成兜底正文，网页地址状态变为 fallback，网页虚拟文件状态仍变为 success。

网页读取过程中的页面反馈：

| 状态 | 用户看到的反馈 | 下一步 |
| --- | --- | --- |
| fetching | 正在抓取网页正文... | 禁用 |
| success | 网页正文已抓取，可进入下一步预览切片 | 启用 |
| fallback | 当前环境无法直接读取网页，已生成可预览的网页正文 | 启用 |
| error | 网页抓取失败，请更换链接 | 禁用 |

网页虚拟文件处理规则：

- 文件名使用网页域名生成，例如 `example.com.html`。
- 文件内容优先使用真实网页正文。
- 抓取失败时使用兜底正文。
- 网页虚拟文件进入 Step 2 的文件下拉列表。
- 网页虚拟文件进入 Step 3 的文件处理进度。

### 6.5 Step 1 到 Step 2

用户点击“下一步”后：

1. 系统检查当前上传方式是否满足进入条件。
2. 若不满足，按钮保持禁用，用户不能点击。
3. 若满足，进入 Step 2。
4. Step 顶部进度条中 Step 1 显示完成态，Step 2 显示当前态。
5. Step 2 根据 Step 1 的上传方式渲染对应规则配置。

进入 Step 2 后保留的数据：

| 数据 | 来源 | 用途 |
| --- | --- | --- |
| 上传方式 | Step 1 选择 | 决定规则配置页面 |
| 上传文件列表 | Step 1 上传 | 预览和处理 |
| 网页虚拟文件 | Step 1 网页读取 | 网页切片预览和处理 |
| 网页正文 / 兜底正文 | 网页抓取结果 | 预览块生成 |

### 6.6 Step 2 文件标签交互

文件标签是 Step 2 的通用能力，所有上传方式都展示。

添加标签：

1. 用户在标签输入框输入内容。
2. 按 Enter 或点击“添加”按钮。
3. 系统去除首尾空格。
4. 若内容为空，不添加。
5. 若标签已存在，不重复添加。
6. 若标签有效，加入已选标签列表。

删除标签：

1. 用户点击已选标签右侧删除按钮。
2. 系统从已选标签列表移除该标签。

推荐标签：

1. 系统展示前 6 个推荐标签。
2. 用户点击推荐标签。
3. 若未添加过，加入已选标签。
4. 若已添加过，按钮置为禁用或不可重复添加。

标签流转：

- Step 2 中添加的标签保存到 `fileUploadTags`。
- Step 3 处理完成后，生成文档记录时将该标签写入每个文档的 `tags` 字段。

### 6.7 Step 2 规则配置切换交互

Step 2 根据上传方式展示不同规则配置。

#### 文本文档 / 音频文件 / 网页读取

这三类使用通用切片配置。

用户点击切片方式卡片时：

1. 当前切片方式高亮。
2. 分段设置区域切换为对应字段。
3. 清空右侧已有预览内容。
4. 右侧预览区域回到空状态。

字段编辑规则：

- 数字输入变更后立即更新配置。
- 开关切换后立即更新配置。
- 下拉选择后立即更新配置。
- 点击“重置”恢复当前切片方式默认值。

点击“预览块”：

1. 系统读取当前选中的成功文件。
2. 文本文档读取文件内容；无内容时使用示例正文。
3. 音频文件使用转写后的文本；当前版本可使用示例转写正文。
4. 网页读取使用抓取正文或兜底正文。
5. 系统根据当前切片方式生成预览块。
6. 右侧展示预览块数量和内容。

#### 表格型知识

表格型知识只展示基础解析。

用户可见内容：

- 配置解析策略
- 基础解析
- 表格内容解析
- 基于规则的表格行和字段提取
- 生成预览按钮

点击“生成预览”：

1. 系统读取当前表格文件。
2. 当前版本生成表格解析预览示例。
3. 右侧预览标题显示“解析预览”。
4. 右侧数量显示“条记录”。

#### 图片文件

图片文件只展示基础解析。

用户可见内容：

- 配置解析策略
- 基础解析
- 手动解析
- 图片文字识别(OCR)
- 说明文案
- 预览块按钮
- 重置按钮

用户切换“手动解析 / 图片文字识别(OCR)”时：

1. 当前选项 radio 选中。
2. 下方说明文案同步变化。
3. 已生成的右侧预览保留或清空均可，推荐清空以避免误解。

点击“预览块”：

1. 系统读取当前图片文件。
2. 根据当前解析方式生成图片问答预览。
3. 右侧标题显示“问答预览”。
4. 右侧数量显示“条问答”。

点击“重置”：

1. 图片解析方式恢复为“手动解析”。
2. 清空预览内容。
3. 右侧回到空状态。

### 6.8 Step 2 右侧预览交互

右侧预览区域根据上传方式变化。

| 上传方式 | 预览标题 | 数量单位 | 空状态文案 |
| --- | --- | --- | --- |
| 文本文档 | 切片预览 | 预估块 | 点击左侧的“预览块”按钮来加载预览 |
| 表格型知识 | 解析预览 | 条记录 | 点击左侧的“生成预览”查看表格解析效果 |
| 图片文件 | 问答预览 | 条问答 | 点击左侧的“预览块”查看图片问答效果 |
| 音频文件 | 切片预览 | 预估块 | 点击左侧的“预览块”按钮来加载预览 |
| 网页读取 | 切片预览 | 预估块 | 点击左侧的“预览块”按钮来加载预览 |

文件选择下拉：

- 展示所有 status 为 success 的上传文件。
- 网页读取展示网页虚拟文件。
- 切换文件后清空当前预览内容。
- 若无成功文件，展示“暂无已上传文件”。

预览块展示：

- 展示序号。
- 展示字符数。
- 展示内容摘要。
- 内容过长时折行或截断展示。

### 6.9 Step 2 到 Step 3

用户点击 Step 2 底部“下一步”后：

1. 系统进入 Step 3。
2. importStatus 设置为 importing。
3. importProgress 初始化为文件解析中。
4. 每个成功文件初始化处理进度。
5. 文件开始模拟或真实处理。

进入 Step 3 的条件：

- Step 1 已有成功文件或网页虚拟文件。
- Step 2 不强制要求用户生成预览。
- Step 2 标签可为空。
- Step 2 使用当前配置值进入处理。

### 6.10 Step 3 文件处理交互

Step 3 展示处理完成前后的状态。

处理中：

1. 顶部标题显示“正在处理文件”。
2. 显示当前处理阶段文案。
3. 文件列表展示实时进度。
4. 底部不展示“前往文档”，或按钮不可用。

处理完成：

1. 顶部标题显示“处理完成”。
2. 展示“全部文件处理完成”。
3. 文件进度为 100%。
4. 底部按钮变为“前往文档”。
5. 点击后关闭上传流程并返回文档列表。

文件处理阶段文案：

| 进度范围 | 阶段 | 文案 |
| --- | --- | --- |
| 0-33% | cleaning | 文本清洗中 |
| 34-63% | chunking | 规则解析 / 切片处理中 |
| 64-85% | vectorizing | 索引生成中 |
| 86-100% | writing | 写入知识库中 |

### 6.11 Step 3 规则摘要交互

Step 3 顶部展示规则摘要卡片。该卡片必须根据上传方式变化。

| 上传方式 | 第一张摘要卡片标题 | 主标题 | 说明 |
| --- | --- | --- | --- |
| 文本文档 | 分段模式 | 当前切片方式 | 当前切片参数 |
| 表格型知识 | 解析规则 | 表格内容解析 | 基础解析 / 按表格行和字段生成知识条目 |
| 图片文件 | 解析规则 | 手动解析 或 图片文字识别(OCR) | 当前图片解析方式说明 |
| 音频文件 | 转写切片 | 当前切片方式 | 音频转写后处理 / 当前切片参数 |
| 网页读取 | 网页切片 | 当前切片方式 | 网页正文解析后处理 / 当前切片参数 |

第二张摘要卡片展示索引方式。

第三张摘要卡片展示检索设置。

### 6.12 完成后返回文档列表

用户点击“前往文档”：

1. 系统关闭上传流程。
2. 清空上传流程临时状态。
3. 清空上传文件、网页地址、标签输入、处理进度。
4. 文档列表刷新。
5. 新上传文档展示在列表顶部。
6. 新文档带有 Step 2 添加的标签。

## 7. Step 1 上传文件

### 7.1 上传方式

| 上传方式 | 说明 | 支持格式 / 输入 |
| --- | --- | --- |
| 导入文本文档 | 本地上传文档类知识 | `.doc`、`.txt`、`.docx`、`.pdf`、`.ppt`、`.pptx`、`.md` |
| 导入表格型知识 | 表格结构化知识 | `.xlsx`、`.xls`、`.csv`、`.json` |
| 导入图片文件 | 图片内容识别或图片问答 | `.png`、`.jpg`、`.jpeg`、`.bmp` |
| 导入音频文件 | 语音转写后入库 | `.wav`、`.mp3`、`.m4a`、`.amr` |
| 读取网页数据 | 输入网页 URL 读取正文 | 单个 URL |

### 7.2 本地文件上传规则

适用上传方式：

- 文本文档
- 表格型知识
- 图片文件
- 音频文件

交互规则：

- 支持拖拽上传。
- 支持点击上传。
- 支持多文件上传。
- 文件上传后进入文件列表。
- 文件列表展示文件名、格式、大小、进度、状态。
- 支持移除单个文件。
- 支持分页展示。
- 支持只查看问题文件。

文件状态：

| 状态 | 说明 |
| --- | --- |
| uploading | 上传中，显示进度 |
| success | 上传成功，可进入下一步 |
| error | 上传失败，阻止进入下一步 |

下一步按钮规则：

- 文件数量为 0：禁用。
- 存在上传中：禁用。
- 存在失败文件：禁用。
- 至少一个文件成功且无上传中/失败：启用。

### 7.3 网页读取规则

适用上传方式：

- 读取网页数据

字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 网页地址 | URL 输入框 | 是 | 输入需要解析的网页 URL |
| URL 去重 | checkbox | 默认开启 | 自动跳过知识库内已存在的相同网页 |
| 限定解析范围 | checkbox | 可选 | 开启后可填写 CSS 选择器，仅解析正文区域 |
| CSS 选择器 | text | 条件必填 | 限定解析范围开启时展示 |
| 保留超链接 | checkbox | 默认开启 | 保留正文中的链接来源 |

添加链接交互：

1. 用户输入 URL。
2. 点击“添加链接”。
3. 系统创建一条网页虚拟文件。
4. 系统尝试抓取网页 HTML 正文。
5. 若抓取成功，网页虚拟文件状态变为 success。
6. 若抓取失败，系统生成兜底网页正文，网页虚拟文件仍标记为 success。
7. 用户可进入 Step 2 查看切片预览。

网页抓取状态：

| 状态 | 展示文案 | 后续动作 |
| --- | --- | --- |
| fetching | 正在抓取网页正文 | 下一步禁用 |
| success | 网页正文已抓取，可进入下一步预览切片 | 下一步启用 |
| fallback | 当前环境无法直接读取网页，已生成可预览的网页正文 | 下一步启用 |
| error | 网页抓取失败，请更换链接 | 下一步禁用 |

兜底规则：

- 由于浏览器跨域、目标站点权限、登录限制、搜索结果页限制等原因，前端可能无法直接抓取完整网页。
- 若抓取失败，系统必须生成一份可用于预览的网页正文。
- 兜底正文需要包含：
  - 网页来源地址
  - 网页内容摘要
  - 解析结果说明
  - 常见问题示例
- 兜底正文也作为网页虚拟文件进入 Step 2 和 Step 3。

网页下一步按钮规则：

- 未添加 URL：禁用。
- 网页虚拟文件 fetching：禁用。
- 网页虚拟文件 success / fallback：启用。
- 网页虚拟文件 error：禁用。

## 8. Step 2 规则配置

### 8.1 通用区域

Step 2 左侧顶部展示当前上传方式摘要。

字段：

| 字段 | 说明 |
| --- | --- |
| 上传方式标题 | 例如“导入文本文档”“导入表格型知识” |
| 上传方式描述 | 简述当前类型入库逻辑 |

### 8.2 文件标签

所有上传方式都需要展示文件标签配置。

目的：

- 给本次上传的文件添加业务标签。
- 便于后续分类、筛选、检索过滤和召回增强。

字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 标签输入框 | text | 否 | 输入标签后按 Enter 添加 |
| 添加按钮 | button | 否 | 将输入框内容添加为标签 |
| 已选标签 | tag list | 否 | 展示当前已添加标签 |
| 推荐标签 | button list | 否 | 快捷添加常用标签 |

交互规则：

- 输入标签后按 Enter 可添加。
- 点击“添加”按钮可添加。
- 空标签不可添加。
- 重复标签不可重复添加。
- 已选标签可删除。
- 推荐标签点击后加入已选标签。
- 已添加的推荐标签不可重复点击。
- 标签保存到上传文件的 `tags` 字段。

### 8.3 文本文档规则配置

文本文档使用通用切片配置。

切片方式：

| 切片方式 | 说明 |
| --- | --- |
| 智能切片 | 自动识别结构，普通文档默认使用 |
| 按标题切分 | 根据标题层级切分，适合结构化文档 |
| 按问答切分 | 按问答或记录切分，适合 FAQ 和条目型文本 |
| 按页切分 | 按页码边界切分，适合 PDF、报告、合同等 |

#### 智能切片字段

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 结构识别 | switch | 开启 | 自动识别标题、段落、列表等结构 |
| 分段最大长度 | number | 1000 chars | 单个切片最大字符数 |
| 分段重叠长度 | number | 80 chars | 相邻切片保留重复内容 |

#### 按标题切分字段

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 标题层级深度 | select | H2 / H3 | 控制切到几级标题 |
| 分段最大长度 | number | 1500 chars | 标题块最大字符数 |
| 分段重叠长度 | number | 100 chars | 标题块之间保留上下文 |

#### 按问答切分字段

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 记录单位 | select | 自动识别 | 自动识别问答对、表格行、记录 |
| 分段最大长度 | number | 1200 chars | 单条记录最大字符数 |
| 分段重叠长度 | number | 20 chars | 问答记录通常不需要大重叠 |

#### 按页切分字段

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 切分单位 | select | 每页一块 | 按页或每 2 页合并 |
| 保留页码 | switch | 开启 | 保留来源页码 |
| OCR 识别 | select | 自动 | 扫描件或图片型 PDF 自动识别 |

操作按钮：

- 预览块：根据当前文件和规则生成切片预览。
- 重置：恢复当前切片方式默认配置。

### 8.4 表格型知识规则配置

表格型知识不展示通用切片方式。

展示内容：

| 区域 | 内容 |
| --- | --- |
| 配置解析策略 | 表格型知识按行和字段生成知识条目 |
| 基础解析 | 表格内容解析 |
| 说明文案 | 基于规则的表格行和字段提取 |

操作按钮：

- 生成预览：生成表格解析预览。

当前版本不展示以下高级字段：

- 表头所在行
- 记录单位
- 问题字段
- 答案字段

### 8.5 图片文件规则配置

图片文件不展示通用切片方式。

展示内容：

| 区域 | 内容 |
| --- | --- |
| 配置解析策略 | 图片可按人工问答描述或 OCR 文字识别入库 |
| 基础解析 | 手动解析、图片文字识别(OCR) |
| 说明文案 | 手动解析：导入后需人工添加描述信息，以触发解析；OCR：识别图片中的文字，适用于文档扫描件等 |

操作按钮：

- 预览块：生成图片问答预览。

当前版本不展示以下字段：

### 8.6 音频文件规则配置

音频文件先转写为文本，再进入通用切片配置。

- Step 2 显示四种切片方式。
- 描述文案提示“音频转写后选择文本切片策略”。
- 字段与文本文档一致。

处理逻辑：

1. 音频上传成功。
2. 系统进入转写。
3. 转写文本进入切片配置。
4. 按所选规则生成知识片段。

### 8.7 网页读取规则配置

展示逻辑：

- Step 2 显示四种切片方式。
- 描述文案提示“网页正文抓取后选择切片策略”。
- 字段与文本文档一致。

预览逻辑：

- 如果网页抓取成功，使用真实网页正文生成切片预览。
- 如果抓取失败，使用兜底网页正文生成切片预览。
- 用户始终可以看到预览内容。

## 9. Step 3 文件处理

### 9.1 处理状态

顶部展示处理状态：

| 状态 | 展示 |
| --- | --- |
| importing | 正在处理文件 |
| success | 处理完成 |
| error | 处理失败 |

处理阶段：

| 阶段 | 说明 |

### 9.2 规则摘要

Step 3 需要按上传方式展示不同摘要，不能统一显示“智能切片”。

| 上传方式 | 摘要标题 | 摘要内容 |
| --- | --- | --- |
| 文本文档 | 分段模式 | 当前切片方式和切片参数 |
| 表格型知识 | 解析规则 | 表格内容解析 / 基础解析 / 按表格行和字段生成知识条目 |
| 图片文件 | 解析规则 | 手动解析或图片文字识别(OCR) |
| 音频文件 | 转写切片 | 音频转写后处理 / 当前切片方式 |
| 网页读取 | 网页切片 | 网页正文解析后处理 / 当前切片方式 |

### 9.3 文件处理进度

展示内容：

| 字段 | 说明 |
| --- | --- |
| 文件名 | 上传文件名或网页虚拟文件名 |

| 处理进度 | 百分比进度条 |
| 文件数量 | 当前处理文件总数 |

处理完成后：

- 生成文档记录。
- 写入当前知识库文档列表。
- 文档继承上传标签。
- 文档状态为 Ready。
- 更新知识库文档数量。
- 显示成功提示。

网页虚拟文件字段：

| 字段 | 说明 |
| --- | --- |

| name | 域名 `.html` |

## 10. 异常与边界规则

### 10.1 本地文件异常

| 场景 | 处理 |
| --- | --- |
| 文件上传中 | 下一步禁用 |
| 存在失败文件 | 下一步禁用 |
| 文件为空 | 下一步禁用 |
| 文件可删除 | 删除后重新计算下一步可用状态 |

### 10.2 网页抓取异常

| 场景 | 处理 |
| --- | --- |
| URL 为空 | 提示请输入网页链接 |
| 目标站点跨域 | 生成兜底正文 |
| 目标站点不可访问 | 生成兜底正文 |
| 搜索结果页限制抓取 | 生成兜底正文 |
| 抓取中 | 下一步禁用 |
| 已有正文或兜底正文 | 下一步启用 |

### 10.3 标签异常

| 场景 | 处理 |
| --- | --- |
| 输入空标签 | 不添加 |
| 重复标签 | 不重复添加 |
| 删除标签 | 从本次上传标签中移除 |

## 12. 验收标准

### 12.1 Step 1

- 可以选择 5 种上传方式。
- 本地文件上传成功后可进入下一步。
- 上传中或上传失败时不能进入下一步。
- 网页地址添加后会生成网页虚拟文件。
- 网页抓取失败时仍能生成兜底正文。
- 网页虚拟文件成功后可进入下一步。

### 12.2 Step 2

- 所有上传方式都展示文件标签配置。
- 文本文档展示四种切片方式。
- 表格型知识只展示基础解析和表格内容解析，不展示冗余字段。
- 图片文件只展示基础解析、手动解析、图片文字识别(OCR)，不展示冗余字段。
- 音频文件展示转写后的四种切片方式。
- 网页读取展示网页正文后的四种切片方式。
- 点击预览/生成预览后右侧有内容。
- 网页抓取失败时，切片预览仍有内容。

### 12.3 Step 3

- 文件处理进度正常展示。
- 表格导入摘要显示表格解析规则，而不是智能切片。
- 图片导入摘要显示图片解析规则，而不是智能切片。
- 音频导入摘要显示转写切片规则。
- 网页导入摘要显示网页切片规则。
- 处理完成后可生成文档记录并回到文档列表。

## 13. 后续迭代建议

- 表格型知识增加高级配置入口：表头行、问题字段、答案字段、元数据字段。
- 图片文件增加批量人工描述录入和 OCR 结果编辑。
- 网页读取增加后端抓取服务，解决前端 CORS 限制。
- 上传任务增加可恢复、重试、取消能力。
- Step 3 增加失败文件详情和重新处理入口。
- 标签支持从知识库标签池中搜索选择。
